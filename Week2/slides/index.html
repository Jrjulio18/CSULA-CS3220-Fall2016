What happens when you type in an URL in your browser?

The URL

You started your web browser (Chrome, Firefox, etc), and you typed in the web address, or URL, in the address bar.

  • URL stands for Uniform Resource Locator, commonly known as web address
  • A Reference to a web resource and how to retrieve it

The Protocol

  • Called a Uniform Resource Identifier (URI) Scheme
  • Describes what protol to retrieve the web resource (how to get your stuff)
  • Common Web Protocols:
    • http://
    • https:// (encrypted version of http)
  • Other protocols you will use in this class:
    • ssh
    • ftp/sftp
    • file

The Domain Name

  • String identifier used to identify ip addresses
  • The suffix indicates the top level domain(TLD) it belongs to:
    • .org, .com, .edu, .io, etc
  • An IP address is a unique string that identifies devices over a network to allow communication
  • A Domain Name System(DNS) server translates domain names to IP addresses
  • For example, Google offers public DNS servers at 8.8.8.8 and 8.8.4.4
  • The DNS resolutions are cached on your computer for faster access
  • Domains are public information. Example: https://whois.icann.org/en/lookup?name=calstatela.edu

The Resource Path

  • Similar to how folders/directories work on your computer
  • Can be pointed to an actual directory or simply a software defined path

    • The Query Parameter

  • Extra parameters provided to the web server (more on that later)
  • We will use them more and more as the semester goes on.

    The Request and Response

    So you typed in the URL in the browser address bar, press enter. The domain is resolved to an IP address, what happens now?

    The Request-Respond Model

    Your browser, running on your computer, creates a connection with the server (can be a physical computer or a software), and makes a HTTP request.

    The server returns a HTTP response. your browser interprets the response renders the response.
